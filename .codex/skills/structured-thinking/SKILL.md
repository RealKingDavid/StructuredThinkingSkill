# Structured Thinking Skill

Transform any idea — no matter how vague or complex — into a clear, phased plan with granular, executable tasks.

## Your Job

1. **Identify the idea type** from what the user shares (or ask if unclear)
2. **Ask a few focused clarifying questions** to understand scope, constraints, and goals
3. **Generate a tailored framework** with phases → tasks → subtasks
4. **Surface key risks and decisions** the user should be aware of
5. **Recommend immediate next steps** to build momentum

---

## Step 1: Clarify Before Planning

Before generating the framework, ask 2–4 short questions to understand:
- What is the core goal or outcome?
- Who is it for (audience, users, stakeholders)?
- What constraints exist (time, budget, team size, technical skill)?
- What have they already done or decided?

Keep questions tight — don't overwhelm. If the user has already shared enough context, skip straight to the framework.

---

## Step 2: Identify Idea Type

Classify the idea into one of these types and use the matching framework from the references folder:

| Type | Trigger signals |
|------|----------------|
| **Business / Startup** | launching a company, product-market fit, monetization, go-to-market |
| **Product / UX Design** | app, interface, user flow, feature, redesign, prototype |
| **Creative Project** | writing, film, game, art, music, content series |
| **Technical / Engineering** | system architecture, software build, API, infrastructure, algorithm |

If an idea spans multiple types (e.g., a startup building a tech product), choose the **primary** type and pull in notes from the secondary type.

→ Read the relevant reference file before generating the framework:
- Business/Startup: `references/business-startup.md`
- Product/UX Design: `references/product-ux.md`
- Creative Project: `references/creative.md`
- Technical/Engineering: `references/technical.md`

## Step 3: Generate the Framework

Output a structured plan using this format:

```
# [Idea Name] — Structured Plan

## Overview
[2–3 sentence summary of the goal and approach]

## Idea Type: [Type]

---

## Phase 1: [Phase Name]
**Goal:** [What this phase accomplishes]
**Duration:** [Rough estimate]

### Task 1.1 — [Task Name]
- Subtask: ...
- Subtask: ...
- Subtask: ...

### Task 1.2 — [Task Name]
- Subtask: ...
...

---

## Phase 2: [Phase Name]
...

---

## ⚠️ Key Risks & Decisions
- [Risk or decision 1]
- [Risk or decision 2]
- [Risk or decision 3]

## ✅ Start Here — Your First 3 Actions
1. [Most important immediate action]
2. [Second action]
3. [Third action]
```

### Framework rules:
- **3–5 phases** depending on complexity
- **2–5 tasks per phase**
- **2–5 subtasks per task** — these should be specific and actionable, not vague
- Duration estimates should be honest ranges, not optimistic
- "Start Here" actions should be doable within the first week

---

## Step 4: Offer to Go Deeper

After presenting the framework, offer:
- Dive deeper into any specific phase or task
- Create a timeline or prioritization view
- Identify the single highest-leverage action to focus on first
- Stress-test the plan by poking holes in it

---

## Tone & Style

- Be direct and concrete — no filler or generic advice
- Adapt vocabulary to the user's domain (don't use startup jargon for a creative project)
- Be honest about difficulty and effort — don't sugarcoat timelines
- Treat the user as a capable adult who can handle real complexity
