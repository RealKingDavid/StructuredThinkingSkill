# Business / Startup Framework

## Phases

### Phase 1: Validate the Idea
Confirm there's a real problem worth solving before building anything.
- Tasks: Problem interviews, competitor analysis, define target customer, validate willingness to pay
- Subtasks: Write problem hypothesis, recruit 5–10 interviewees, build interview script, synthesize findings, define your riskiest assumption

### Phase 2: Define the Solution
Translate validated insights into a concrete offering.
- Tasks: Define value proposition, scope MVP, map business model, identify key metrics
- Subtasks: Write one-sentence value prop, list MVP features vs. nice-to-haves, choose revenue model, define North Star metric, sketch pricing tiers

### Phase 3: Build & Test
Create the minimum version that lets you learn.
- Tasks: Build or prototype MVP, set up core infrastructure, recruit beta users, run first experiments
- Subtasks: Define MVP acceptance criteria, set up analytics, recruit 10–20 beta users, document feedback loops, run A/B or smoke tests

### Phase 4: Go to Market
Get the product in front of real paying customers.
- Tasks: Define GTM strategy, build distribution channels, craft messaging, launch
- Subtasks: Choose primary acquisition channel, write landing page copy, set up CRM, plan launch sequence, define success metrics for launch

### Phase 5: Scale & Iterate
Double down on what works and build repeatability.
- Tasks: Optimize unit economics, expand channels, build team, systematize operations
- Subtasks: Calculate CAC/LTV, identify top-performing channels, define hiring plan, document playbooks, set quarterly OKRs

## Key Risks to Surface
- Assuming the problem exists without talking to customers
- Building too much before validating
- Choosing the wrong monetization model early
- Underestimating time to first revenue
- Conflating interest with willingness to pay

## Type-specific notes
- Emphasize customer discovery in Phase 1 — most startup failures are premature building
- Always ask: "Who is the beachhead customer?" — the most specific, reachable first audience
- For B2B: sales cycle and procurement complexity matter early
- For consumer: distribution and virality loops are critical
